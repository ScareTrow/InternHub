# Contributing to InternHub

Thanks for contributing to InternHub. Keep changes focused, readable, and aligned with the existing project structure.

## Branches

- Use `feature/*` for new functionality.
- Use `fix/*` for bug fixes.
- Branch from the current main development branch and keep branch names short and descriptive.

## Commits

- Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, and `chore:`.
- Write commit messages in the imperative mood.
- Keep unrelated changes in separate commits when possible.

## Pull Requests

- Open pull requests with a short summary of the change and the reason for it.
- Reference related issues when they exist.
- Include testing or validation notes for the affected area.
- Request review before merge and address feedback with follow-up commits.

## Code Expectations

- Prefer readable code over clever code.
- Follow the existing folder structure, naming, and patterns used in the repository.
- Remove debug code, dead code, and temporary files before submitting.
- Keep formatting changes scoped to the files you actually modify.
- Update documentation when setup, behavior, or APIs change.
